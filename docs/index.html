Think of it as having a research assistant that can instantly search millions of articles, find related papers, and summarize findings for you. Instead of clicking through web pages, you type simple commands and get structured results you can immediately use.

Why Use a Command-Line Tool?

You might be wondering: "Why would I type commands when I can just use the PubMed website?" Fair question. Here's why researchers find command-line tools valuable:

Speed. Once you learn the basic commands, searching becomes nearly instant. No waiting for pages to load, no clicking through menus. Type a command, get results.

Reproducibility. Commands can be saved, shared, and re-run. If a colleague asks "how did you find those papers?", you can share the exact command. This matters for systematic reviews and meta-analyses where search methodology must be documented.

Automation. Need to run the same search every week? Commands can be scheduled to run automatically. Need to search for 50 different terms? Write a simple script instead of clicking 50 times.

Integration. Results come as structured data (JSON) that can feed directly into other tools â€” reference managers, analysis scripts, or AI assistants.

Installation

Before we begin, let's get the tool installed on your computer. Don't worry if you've never installed software this way â€” we'll go step by step.

1 Open Terminal

The Terminal is an application on your Mac that lets you type commands. Think of it as a text-based way to control your computer.

To open Terminal:

  • Press Cmd + Space to open Spotlight
  • Type "Terminal" and press Enter

You'll see a window with a blinking cursor. This is where you'll type commands. When we show commands in gray boxes below, you'll type them here and press Enter.

2 Install Homebrew (if needed)

Homebrew is a tool that makes installing software on Mac easy. It's like an app store for command-line tools. Many researchers use it to install scientific software, programming languages, and utilities.

First, check if you already have Homebrew by typing:

brew --version

If you see a version number (like "Homebrew 4.2.0"), you're all set â€” skip to Step 3.

If you see "command not found", install Homebrew by pasting this command:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

This will ask for your password (the one you use to log into your Mac) and take a few minutes. The password won't show as you type â€” that's normal for security. Just type it and press Enter.

3 Install pubmed-cli

Now we can install pubmed-cli with two commands:

brew tap henrybloomingdale/tools
brew install pubmed-cli

The first command tells Homebrew where to find our tool. The second installs it.

4 Verify Installation

pubmed --help

You should see a list of available commands. If you do, congratulations â€” you're ready to go!

PubMed allows 3 searches per second without a key, and 10 per second with one. For casual use, you don't need a key. For heavy use, get a free key at ncbi.nlm.nih.gov/account.

Once you have a key, set it in Terminal:

export NCBI_API_KEY="your-key-here"

Your First Search

Let's search for articles about a topic. The basic format is:

pubmed search "your search terms"

For example, to search for articles about fragile X syndrome:

Understanding the Flags

Flags are options that modify how a command works. They start with -- (two dashes).

  • --limit 5 â€” Show only 5 results instead of the default 20
  • --human â€” Format output as a readable table

Fetching Article Details

Once you find an interesting PMID, you can get full details including the abstract. The fetch command retrieves complete article information: 

pubmed fetch 41651254 --human --full

This shows:

  • Full title
  • All authors with affiliations
  • Journal name and publication date
  • DOI (digital object identifier)
  • Complete abstract

You can fetch multiple articles at once by separating PMIDs with commas:

pubmed fetch 41651254,41648852,41645177 --human

Filtering Your Searches

Real research often requires more specific searches. Here's how to narrow results:

Filter by Year

pubmed search "autism therapy" --year 2023-2025 --limit 10 --human

Filter by Publication Type

# Only review articles
pubmed search "diabetes treatment" --type review --limit 10 --human

# Only clinical trials
pubmed search "SSRI depression" --type trial --limit 10 --human

# Only meta-analyses
pubmed search "statin cardiovascular" --type meta-analysis --limit 10 --human

Sort Results

# Most recent first
pubmed search "CRISPR" --sort date --limit 10 --human

# Most cited first
pubmed search "CRISPR" --sort cited --limit 10 --human

Combine Filters

pubmed search "COVID vaccine" --year 2024-2025 --type trial --sort date --limit 10 --human

Exporting to Files

For systematic reviews or when you need to share results, export to files:

CSV (for Excel/Spreadsheets)

pubmed search "autism biomarkers" --limit 100 --csv results.csv

Open results.csv in Excel to see a spreadsheet with PMID, Title, Year, Journal, DOI, and Type.

JSON (for Programming/Analysis)

pubmed search "autism biomarkers" --limit 100 --json > results.json

JSON is a structured format that programming languages and AI tools can easily read.

Finding Related Articles

PubMed tracks relationships between articles. Use these to expand your literature search:

Who Cited This Article?

pubmed cited-by 38000001 --limit 10 --human

Shows newer articles that reference this one â€” great for seeing how research has evolved.

What Does This Article Cite?

pubmed references 38000001 --limit 10 --human

Shows the article's bibliography â€” useful for finding foundational papers.

Similar Articles

pubmed related 38000001 --limit 10 --human

PubMed's algorithm finds articles on similar topics, even if they don't directly cite each other.

Looking Up MeSH Terms

MeSH (Medical Subject Headings) is PubMed's controlled vocabulary. Using MeSH terms makes searches more precise. Look up terms like this: 

pubmed mesh "depression"
Output
{
  "ui": "D003863",
  "name": "Depression",
  "scope_note": "Depressive states usually of moderate intensity...",
  "tree_numbers": ["F01.145.126.350", "F01.470.282"],
  "entry_terms": ["Depressive Symptoms", "Emotional Depression", ...]
}

The entry_terms show synonyms that PubMed recognizes. The tree_numbers show where this term sits in MeSH's hierarchy.

Literature Synthesis

Literature synthesis uses AI to read papers and generate summaries. This requires either the Claude CLI (free with a Claude account) or an OpenAI API key.

The synth command searches PubMed, scores papers for relevance, and generates a narrative synthesis with proper citations: 

pubmed synth "FMRP function in synaptic plasticity" --papers 5 --words 200 --claude
Output

*Searched 30 papers, scored 30, used 5*

   organization... Molecular psychiatry. [PMID: 40931164]

Export Options

# Word document
pubmed synth "ketamine for depression" --papers 5 --docx review.docx --claude

# BibTeX (for LaTeX)
pubmed synth "ketamine for depression" --papers 5 --bibtex refs.bib --claude

# RIS (for EndNote/Zotero)
pubmed synth "ketamine for depression" --papers 5 --ris refs.ris --claude

Answering Research Questions

The qa command answers yes/no research questions, automatically deciding whether to use its training knowledge or search PubMed: 

pubmed qa "Does FMRP regulate synaptic plasticity?" --claude
